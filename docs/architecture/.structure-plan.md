## Migration Plan

### Phase 1: Create New Structure (5 min)
1. Create `skills/` directory
2. Create `internal-docs/` directory
3. Create `scripts/` directory (already exists, just move files)

### Phase 2: Move Files (10 min)
4. Move validator scripts → `skills/validator/`
5. Move status reports → `internal-docs/build-logs/`
6. Move design docs → `internal-docs/design-decisions/`
7. Move utility scripts → `scripts/`
8. Keep only essential docs at root

### Phase 3: Update Links (5 min)
9. Update README.md paths
10. Update VALIDATOR_SKILL.md → skills/validator/SKILL.md
11. Update scripts to use new paths

### Phase 4: Commit & Document (5 min)
12. Git commit with clear migration message
13. Update docs/README.md as index
14. Update skills/README.md as agent entry point

## Benefits

**For Agents:**
- ✅ Clear entry point: `skills/` directory
- ✅ One skill per capability: validator, developer, trader
- ✅ Self-contained: Each skill has scripts + docs

**For Developers:**
- ✅ Clean root: Only README, LICENSE, Cargo.toml
- ✅ Clear structure: docs/ for users, internal-docs/ for history
- ✅ Easy navigation: Find what you need fast

**For Maintenance:**
- ✅ Scalable: Add new skills without cluttering root
- ✅ Organized: Status reports archived, not deleted
- ✅ Professional: Looks like a mature project
