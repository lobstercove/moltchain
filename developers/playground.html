Programs Playground

The MoltChain Programs Playground is a fully browser-based IDE for writing, compiling, and deploying smart contracts — no local toolchain required. Jump straight into building without installing Rust, Docker, or the CLI.

Launch Now
Open the Programs Playground in a new tab to follow along with this guide.

Overview

The Playground provides a complete contract development environment inside your browser:

  • Write — Syntax-highlighted Rust editor with autocomplete and inline error markers
  • Compile — One-click WASM build via the cloud compiler service (no local Rust needed)
  • Deploy — Push compiled programs directly to testnet or mainnet
  • Interact — Call deployed contract methods and inspect return values

It's ideal for prototyping, learning, and quick experiments. For production workflows, pair it with the CLI and local contract toolchain.

Interface Guide

The Playground is divided into four main areas:

1
Code Editor (Left Panel)
The main editing area where you write your Rust smart contract. Supports syntax highlighting, bracket matching, and code folding. Use Ctrl+S / ⌘S to trigger a compile.
2
Output Console (Bottom)
Displays compiler output, deployment logs, and method call results. Errors include line numbers that link back to the editor. The console is scrollable and can be resized by dragging its top edge.
3
Deploy Panel (Right)
After a successful compile, this panel shows binary size, estimated deploy cost, and a Deploy button. Once deployed, it switches to an interaction view where you can select methods, enter arguments, and execute calls.
4
Toolbar
Network Selector — Switch between Testnet (recommended for experiments) and Mainnet.
Template Selector — Load one of the built-in example contracts instantly.
Compile / Deploy — Action buttons that drive the workflow.

Built-in Examples

The Playground ships with five starter contracts you can load via the template selector:

Hello World
Bare-minimum contract — a single greet() method that returns a string. Perfect for verifying the compile-deploy loop works end to end.
Token
ERC20-like fungible token with mint, transfer, approve, and balance_of. Demonstrates storage maps and event emission.
NFT
Basic non-fungible token with mint, transfer, and owner_of. Shows unique-ID generation and ownership tracking.
Counter
Simple state management — increment, decrement, and get. Great for learning how on-chain state persistence works.
Escrow
Multi-party escrow with deposit, approve, and release. Demonstrates multi-signer logic and conditional fund transfers.

Build & Deploy Workflow

Deploying a contract from the Playground takes five steps:

1
Write Your Contract
Type or paste Rust code in the editor, or pick a template from the selector dropdown. The editor validates syntax in real time.
2
Compile
Click Compile (or ⌘+Enter). The source is sent to the WASM compiler service and the resulting binary appears in the output console. Errors, if any, are shown with line numbers.
3
Select Network
Use the network selector to choose Testnet (recommended for experiments) or Mainnet. Testnet tokens are free — request them via molt faucet or the in-playground faucet button.
4
Deploy
Click Deploy. The Playground signs a deploy transaction, broadcasts it, and waits for finalization. On success you receive a Program ID and transaction hash.
5
Interact
The deploy panel switches to an interaction view. Select a method from the dropdown, fill in arguments as JSON, and click Call. Read-only calls are free; state-mutating calls require a signature.

Tips & Best Practices

Use #![no_std]
All MoltChain contracts must be #![no_std]. The standard library is not available in the on-chain WASM runtime. The Playground templates already include this attribute.
Test Locally First
For anything beyond quick experiments, write unit tests with cargo test locally before deploying. The SDK's test_mock framework lets you simulate the full on-chain environment offline.
Check Gas Estimates
Before deploying to mainnet, review the estimated deploy cost in the Deploy Panel. Large binaries can be expensive — use wasm-opt -Oz to shrink them. Deploy to testnet first to verify behavior at no cost.
Keyboard Shortcuts
  • ⌘+Enter — Compile
  • ⌘+Shift+Enter — Deploy
  • ⌘+S — Save to browser storage
  • ⌘+K — Search docs
