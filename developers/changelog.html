Changelog

A complete history of every MoltChain release — new features, improvements, breaking changes, and fixes.

v0.3.7 March 18, 2026

Blockchain Alignment Phases 3 & 4 — Archive Mode, Oracle Attestation, EVM Precompiles, Hash Determinism & More

New Features

  • Archive Mode & Historical State Queries — new getAccountAtSlot RPC method returns account state at any past slot. Write-through snapshots stored in dedicated CF_ACCOUNT_SNAPSHOTS column family with pruning support (Task 3.9, M-16)
  • Wire Envelope Format (Payload Type Envelope) — transactions now use a 4-byte header ([0x4D, 0x54, version, type]) followed by bincode payload. Three-format decoder supports V1 envelope, legacy bincode, and JSON for full backward compatibility. Centralized decode_transaction_bytes() replaces 7+ first-byte heuristic decode sites across RPC (Task 3.10, M-6)
  • TOFU Certificate Rotation — new CertRotation P2P gossip message allows validators to rotate TLS certificates without losing peer trust. Validated via old-fingerprint match, cert verification, FP/cert consistency check, and 1-hour rate limit. Successful rotations re-gossipped to all peers (Task 3.11, M-9)
  • Eclipse Defense (AS-Level) — /16 IPv4 and /32 IPv6 prefix bucketing limits peers per autonomous-system bucket to 4, preventing hosting-provider-level eclipse attacks. Zero external dependencies (Task 3.12, M-10)
  • Oracle Multi-Source Attestation — validators submit signed price attestations via system instruction 30; consensus prices derived from BFT-weighted medians stored in CF_STATS (Task 3.6, M-17)
  • EVM Precompiles & eth_getLogs — 9 standard precompiles (ecRecover, SHA-256, RIPEMD-160, identity, modexp, ecAdd, ecMul, ecPairing, blake2f) plus two-phase eth_getLogs with topic/address filtering per EIP-1474 (Task 3.4, M-12)
  • Contract Upgrade Governance (Timelock) — upgrade proposals require governance approval with configurable timelock delay before execution (Task 3.3, M-14)
  • Deterministic BFT Timestamps — block timestamps derived from BFT-weighted median of validator proposals, monotonically increasing (Task 3.2, M-15)

Improvements

  • WASM Memory Limit Increase — contract memory ceiling raised from 1 MiB to 4 MiB with configurable per-contract limits (Task 3.5, M-13)
  • WebSocket Keepalive Standard — 30-second ping/pong heartbeat with strict single-pong timeout; prevents stale connections from accumulating (Task 3.8, M-19)
  • Epoch Boundary Logic — rewards distributed at epoch boundaries using Solana-style epoch model (Task 3.1, M-1)

Fixes

  • Mempool Express Lane Removed — eliminated reputation-based priority multiplier; mempool now uses strict fee-only ordering to prevent MEV favoritism (Task 3.7, M-8)

Phase 4 — Transaction Integrity, Fee Fairness & Storage Metering

New Features

  • Transaction Hash Determinism — Transaction::hash() algorithm formally specified as SHA-256(bincode(message) || sig_0 || sig_1 || ...). New Transaction::message_hash() method returns the signature-independent message digest. RPC getTransaction now includes message_hash field alongside the txid. 8 new wire-format determinism tests (Task 4.1, H-7)
  • Contract Storage Metering — per-byte compute cost on storage_write (1 CU/byte) with 10 MB total storage cap per contract. Live byte tracking via ContractContext.storage_bytes_used; storage_delete reclaims freed bytes. Prevents unbounded state growth (Task 4.3, M-4)
  • IBC Exploration & Roadmap — comprehensive research document at docs/strategy/IBC_EXPLORATION.md covering IBC requirements, existing MoltChain primitives (commit certificates, state proofs, BFT finality), gap analysis, and phased implementation roadmap. Full IBC deferred; MoltBridge adequate for current phase (Task 4.4, H-14)

Fixes

  • Fee Discount MEV Vector Removed — deprecated apply_reputation_fee_discount(); all transactions now pay the flat base fee regardless of MoltyID reputation score. Eliminates identity-based fee manipulation and aligns with standard blockchain fee models (Task 4.2, M-7)

v0.2.17 March 10, 2026

Ghost Validator Fix, CLI Deploy Flags, Poseidon Hash & Token Supply

New Features

  • CLI: --supply <amount> flag sets initial total supply on deploy (auto-converted to shells via decimals)
  • CLI: --metadata <json> flag attaches custom metadata to the symbol registry entry
  • CLI: On-chain verification after deploy — polls getAccountInfo to confirm contract is live
  • WASM runtime: host_poseidon_hash host function — BN254 Poseidon hash for Merkle trees and ZK circuits (2,000 CU)
  • Contract SDK: poseidon_hash(left, right) → [u8; 32] wrapper in moltchain-contract-sdk

Improvements

  • RPC: getContractInfo now falls back to registry metadata for total_supply when on-chain storage key is empty
  • Explorer: Token supply page uses callContract fallback for total_supply view function
  • Validator: Two-tier stale cleanup — never-active ghost validators cleaned after 500 slots, previously-active after 50 epochs
  • Validator: P2P machine fingerprint pre-check rejects new validator announcements from machines that already have a registered identity

Fixes

  • Ghost validator purge on startup — removes validators with 0 blocks and 0 activity, returns bootstrap grants to treasury
  • Website: WS connection indicator no longer blinks on every 5-second stats poll (uses statsInitialLoad flag)
  • Website: resolveValidatorCount() filters ghost validators (last_active_slot=0 AND blocks_produced=0)
  • EU VPS crash loop resolved — RocksDB LOCK held by zombie supervisor process
v0.12.0 February 18, 2026

DEX Hardening, Governance Execution & Prediction Market Fixes

New Features

  • Governance execution dispatch — dex-governance execute_proposal now performs real cross-contract calls (create_pair, update_pair_fees, pause_pair) with full audit trail
  • Reduce-only order flag — order_type | 0x80 in dex-core validates against open margin positions via cross-contract call to dex-margin opcode 26
  • New dex-core opcode 30: set_margin_address — links dex-core to dex-margin for reduce-only validation
  • New dex-margin opcode 26: query_user_open_position — queries open position for a trader on a specific pair
  • MoltyID integration in DEX wallet — .molt name resolution and reputation badge display on connect
  • Batch .molt name resolution for saved wallets via batchReverseMoltNames RPC

Improvements

  • Tokenomics rebalance — reward pool reduced to 100K MOLT/month with epoch cap enforcement; listing liquidity threshold lowered to 10K MOLT
  • Prediction market sell math rewritten — binary search over total_a_for_sets() replaces broken equal-partition heuristic for multi-outcome markets
  • Prediction market return data — create_market, add_liquidity, buy_shares, sell_shares, withdraw_liquidity now write full u64 via set_return_data()

Fixes

  • Oracle safety — close_position, partial_close, remove_margin now reject (codes 5/7) when oracle price is unavailable instead of allowing zero-PnL exits
  • Prediction u32 truncation — call() dispatcher no longer overwrites function-level return_data on success for opcodes 1,3,4,5,13-15
  • CI: all cargo fmt (610+ diffs) and clippy warnings (47 warnings) resolved — CI pipeline fully green

Breaking Changes

  • dex-margin: close_position and partial_close now return error code 5 (previously returned full margin) when oracle is unavailable — agents must handle this new error
  • dex-margin: remove_margin now returns error code 7 when oracle is unavailable — previously skipped health check entirely
  • dex-core: place_order with reduce-only flag (0x80) returns new error code 12 when validation fails
