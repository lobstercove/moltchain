Privacy Architecture

Shield (Deposit)

Transparent → Shielded

Public: amount, commitment
Private: value, blinding
Proves: commitment = Pedersen(value, blind)
~5,000 constraints | <1s proving

Unshield (Withdraw)

Shielded → Transparent

Public: root, nullifier, amount, recipient
Private: note, spending_key, merkle_path
Proves: ownership + membership
~50,000 constraints | <3s proving

Transfer (Private)

Shielded → Shielded

Public: root, nullifiers[], commits[]
Private: notes, keys, paths, outputs
Proves: value conservation + ownership
~200,000 constraints | <5s proving

Cryptographic Primitives

Proving System
Groth16 (trusted setup, 128-byte proofs, ~3ms verification)
Elliptic Curve
BN254 (alt_bn128) — EVM pairing precompile compatible
Commitment Scheme
Pedersen Commitments: C = value × G + blinding × H (hiding + binding)
Hash Function
Poseidon (algebraic, SNARK-friendly, 8x cheaper in-circuit vs SHA-256)
Note Encryption
ECDH key agreement + ChaCha20-Poly1305 (authenticated encryption)
Nullifier Derivation
nullifier = Poseidon(serial, spending_key) — deterministic, non-grinding
Library
arkworks-rs (ark-groth16, ark-bn254, ark-crypto-primitives)

Security Properties

Zero-Knowledge

Proofs reveal nothing about private data. Amounts, senders, and recipients remain hidden.

Soundness

Invalid proofs cannot be forged. The Groth16 proving system is computationally sound under the q-PKE assumption.

Double-Spend Prevention

Each note has a unique nullifier. Once published on-chain, the note cannot be spent again.

Value Conservation

Transfer circuits enforce sum(inputs) == sum(outputs). No MOLT can be created or destroyed in shielded transfers.

Selective Disclosure

Viewing keys can be shared with auditors for compliance. Full privacy with optional transparency.

Proof Malleability Protection

Proofs are bound to the transaction hash via Fiat-Shamir, preventing front-running attacks.
